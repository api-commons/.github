# Contributing to API Commons projects

Thanks for your interest in contributing!

- **Issues** — search first, then open a clear, reproducible issue on the
  relevant repository.
- **Pull requests** — keep them focused; include tests where applicable and make
  sure the project builds (`yarn build`) and tests pass before submitting.
- **Discussions & ideas** — use the repository's issues or
  [apicommons.org](https://apicommons.org).
- **Security** — see [SECURITY.md](./SECURITY.md); do not file public issues for
  vulnerabilities.

By contributing you agree your contributions are licensed under the same license
as the project (Apache-2.0 for the Spotlight projects).
